GETTING STARTED

Installation

Zyppr is available as a single, self-contained binary for macOS, Linux, and Windows. No runtime dependencies (no Node.js, no Python).

macOS / Linux

curl -s https://cdn.zyppr.online/install.sh | bash

Windows (PowerShell)

iwr -useb https://cdn.zyppr.online/install.ps1 | iex

npm (alternative)

npm install -g zyppr

Verify installation:

zyppr version
# → zyppr 1.0.0 (QUIC tunnel client)

Zero-Download Method (SSH)

Don't want to install anything? Zyppr supports standard SSH tunneling — works on every OS since 2005. No download, no signup, no binary needed.

# Free tier (no account needed)
ssh -p 443 -R0:localhost:3000 qr@free.zyppr.online

# With your auth token (Pro features)
ssh -p 443 -R0:localhost:3000 YOUR_TOKEN@tunl.zyppr.xyz

This is a world-first dual architecture — both a CLI binary (QUIC-based) and a zero-download SSH method, running simultaneously.

Authentication

Save your auth token to unlock all Pro features:

zyppr auth YOUR_TOKEN
# → Token saved to ~/.zyppr/config

Get your token from the Dashboard after signing up.

Quick Start

Expose a local web server to the public internet:

# HTTP tunnel on port 3000
zyppr tunnel 3000

# Multiple ports at once
zyppr tunnel 3000 8080

# TCP tunnel (databases, SSH, RDP)
zyppr tunnel 5432 --tcp

# UDP tunnel (game servers, IoT)
zyppr tunnel 25565 --udp

# With basic auth
zyppr tunnel 3000 --auth admin:secret

Output:

Tunnel Status: Online
HTTPS: https://cosmic-name.tunl.zyppr.xyz
HTTP: http://cosmic-name.tunl.zyppr.xyz
Latency: 24ms

Config-as-Code

Define all tunnels in a zyppr.yml file and start everything with zyppr up:

# zyppr.yml
tunnels:
  web:
    port: 3000
    auth: admin:password

  api:
    port: 8080

  db:
    port: 5432
    proto: tcp

  game:
    port: 25565
    proto: udp
zyppr up
# → 4 tunnels started
TRANSPORT METHODS

SSH Transport

Works on any machine with a terminal. No downloads. No signup for the free tier.

# HTTP tunnel
ssh -p 443 -R0:localhost:3000 TOKEN@tunl.zyppr.xyz

# TCP tunnel (add +tcp suffix)
ssh -p 443 -R0:localhost:5432 TOKEN+tcp@tunl.zyppr.xyz

# UDP tunnel (add +udp suffix)
ssh -p 443 -R0:localhost:25565 TOKEN+udp@tunl.zyppr.xyz

Features: Live terminal UI with QR code, real-time traffic stats, color-coded request logs, keybindings (q to quit, h for help).

QUIC Transport (CLI Binary)

The zyppr CLI uses QUIC — a modern UDP-based transport protocol that's faster than TCP. Benefits include:

  • 0-RTT connection establishment
  • Built-in encryption (TLS 1.3)
  • Multiplexed streams (no head-of-line blocking)
  • Auto-reconnect on network changes
zyppr tunnel 3000 --server tunl.zyppr.xyz:4443

Browser Tunnel

Tunnel directly from your web browser — no download, no CLI, no terminal. Navigate to your tunnel URL's /tunnel/ws endpoint, paste your token, and your localhost is live.

Supports: HTTP requests, Server-Sent Events (SSE), chunked transfer encoding, auto-reconnect, live request/byte counters, connection heartbeat.

This is a world-first feature — no other tunneling platform offers browser-based tunnel clients.

UDP Bridge (Zero-Download)

Tunnel UDP traffic without installing any binary. Uses a lightweight Node.js bridge script:

node -e "$(curl -sL tunl.zyppr.xyz/bridge.js)" 25565 YOUR_TOKEN

Use cases: Minecraft servers, DNS forwarding, VoIP, IoT sensor data, any UDP-based protocol.

World-first — no other tunneling service offers zero-download UDP tunneling.

TUNNEL TYPES

HTTP/HTTPS Tunnels

The default tunnel type. Every HTTP tunnel gets automatic HTTPS with a valid TLS certificate.

zyppr tunnel 3000
# → https://cosmic-name.tunl.zyppr.xyz

Headers forwarded: X-Forwarded-For, X-Forwarded-Proto, X-Real-IP. WebSocket upgrade is supported automatically.

TCP Tunnels

Expose any TCP service — databases, SSH, RDP, custom protocols:

zyppr tunnel 5432 --tcp
# → tcp://tunl.zyppr.xyz:54321

# Connect from anywhere:
psql -h tunl.zyppr.xyz -p 54321 -U postgres

UDP Tunnels

Expose UDP services — game servers, DNS, IoT devices:

zyppr tunnel 25565 --udp
# → udp://tunl.zyppr.xyz:25565

Zyppr is the only tunneling platform that supports UDP. ngrok, Cloudflare Tunnel, and Pinggy do not offer UDP tunneling.

Custom Domains + Auto-SSL

Bring your own domain with automatic Let's Encrypt TLS certificates:

# 1. Add a CNAME record:
tunnel.yourdomain.com → CNAME → tunl.zyppr.xyz

# 2. Register via API:
POST /api/v1/custom-domain
{ "domain": "tunnel.yourdomain.com" }

# 3. SSL is provisioned automatically
✓ https://tunnel.yourdomain.com → your localhost

Auto-renewal is included. Certificates are provisioned on first request.

DEVELOPER FEATURES

Request Inspector

Every request hitting your tunnel is captured and available via the API:

GET /api/v1/inspect?token=YOUR_TOKEN

# Returns: method, path, headers, body, status, timing

View captured requests in the Dashboard or via the REST API. Requests are stored in-memory per session.

Request Replay

Replay any captured request — optionally with modifications:

POST /api/v1/replay?token=YOUR_TOKEN
{
  "method": "POST",
  "path": "/api/webhook",
  "headers": { "Content-Type": "application/json" },
  "body": "{\"event\": \"test\"}"
}

Snapshots

Create point-in-time captures of your tunnel's HTTP responses. Persisted to SQLite — survives server restarts.

POST /api/v1/snapshot?token=YOUR_TOKEN
{ "paths": ["/", "/api/users", "/dashboard"] }

# Retrieve snapshot later:
GET /api/v1/snapshots/snap-abc123/serve?path=/api/users

Auto-Swagger

Zyppr observes your live traffic and auto-generates OpenAPI 3.0 documentation. Every unique path, HTTP method, and response code is documented automatically.

GET /api/v1/swagger?token=YOUR_TOKEN

# Returns OpenAPI 3.0 JSON with all observed endpoints

No annotations, no comments, no hand-writing specs. Your API documents itself by usage.

Webhooks

Queue, inspect, and replay incoming webhooks:

# Enable webhook queuing
POST /api/v1/webhooks/toggle?token=YOUR_TOKEN
{ "enabled": true }

# View queued webhooks
GET /api/v1/webhooks?token=YOUR_TOKEN

# Replay a webhook
POST /api/v1/webhooks/replay?token=YOUR_TOKEN
{ "id": "wh-abc123" }

Webhook Breakpoints

Pause an incoming request mid-flight. Inspect it. Modify it. Then release it to your server. Like a debugger breakpoint, but for HTTP traffic.

# Enable breakpoints
POST /api/v1/webhooks/breakpoints?token=YOUR_TOKEN
{ "enabled": true }

# Paused requests auto-release after 60 seconds

Exclusive to Zyppr — no other tunneling platform offers request breakpoints.

DB Studio

A built-in web SQL browser for your tunneled databases. Browse schemas, explore tables, and run read-only queries — all from your dashboard.

# Connect to a tunneled database
POST /api/v1/db-studio/connect?token=YOUR_TOKEN
{ "driver": "postgres", "dsn": "postgres://user:pass@localhost:5432/mydb" }

# Browse schema
GET /api/v1/db-studio/schema?token=YOUR_TOKEN

# Run a query (read-only)
POST /api/v1/db-studio/query?token=YOUR_TOKEN
{ "sql": "SELECT * FROM users LIMIT 10" }

Supports PostgreSQL, MySQL, and SQLite. World-first in tunneling platforms.

SECURITY

E2E Encryption

Every tunnel session generates a unique cryptographic fingerprint displayed as emoji (🚀🌊💎🔑). Compare them to verify no MITM interception.

GET /api/v1/e2e/verify?token=YOUR_TOKEN

# Returns: { "fingerprint": "🚀🌊💎🔑", "hex": "C768:6856:56E4:E41C" }

AI Gateway

Real-time threat detection on your tunnel traffic. Automatically detects and blocks:

  • SQL Injection attacks
  • Cross-Site Scripting (XSS)
  • Path traversal (../../etc/passwd)
  • Suspicious user agents (scanners, bots)
  • Command injection attempts
GET /api/v1/ai-gateway/stats?token=YOUR_TOKEN

# Returns threat summary, blocked count, classifications

World-first — no other tunneling platform has built-in AI threat detection.

Rate Limiting

Protect your tunnel with per-session request limits:

POST /api/v1/rate-limit?token=YOUR_TOKEN
{ "requests_per_minute": 60, "burst": 10 }

Header Rules

Add, remove, or modify response headers without touching your backend:

POST /api/v1/header-rules?token=YOUR_TOKEN
{
  "action": "set",
  "header": "Access-Control-Allow-Origin",
  "value": "*"
}

Common use: CORS headers, security headers (X-Frame-Options, CSP), cache control.

  "name": "Redirect old paths",
  "path": "/old-page",

# Available actions: redirect, block, mock, modify

Edge Mesh

Create private networks linking multiple developer laptops. Each member's tunnel gets a .zyppr.local hostname:

POST /api/v1/mesh/join
{ "mesh_id": "team-alpha", "hostname": "backend" }

# Now other team members can access:
curl http://backend.zyppr.local/api/users

# View the mesh topology:
GET /api/v1/mesh/topology?mesh_id=team-alpha

Includes health tracking (online/offline/degraded), automatic stale member cleanup, and hostname resolution. World-first in tunneling.

Chaos Engineering

Inject faults into your tunnel to test your app's resilience:

POST /api/v1/chaos?token=YOUR_TOKEN
{
  "latency_ms": 500,
  "error_rate": 0.05,
  "error_code": 500
}

# → 500ms added latency + 5% of requests return 500

Test timeouts, retry logic, circuit breakers, and fallback behavior before your users do. Exclusive to Zyppr.

P2P Mesh

Connect two machines directly via UDP hole punching — bypassing the server entirely:

# On Machine A:
zyppr p2p --peer my-peer-id --local-port 8080

# On Machine B:
zyppr p2p --peer my-peer-id --local-port 9090

# → Direct UDP connection established
# → Falls back to server relay if hole punching fails

The CLI registers with the signaling server, exchanges NAT info, and punches through firewalls. If direct P2P fails, it automatically falls back to a TURN-like server relay. World-first in tunneling platforms.

Tunnel Sharing

Share tunnel access with teammates via secure, expiring links:

POST /api/v1/share/create?token=YOUR_TOKEN
{ "access": "read", "ttl": "24h" }

# → Share link: https://tunl.zyppr.xyz/s/abc123
# Max 10 active shares per session

Health Checks

Automatic uptime monitoring with branded status pages:

POST /api/v1/health/config?token=YOUR_TOKEN
{ "path": "/health", "interval": 60 }

# Branded status page:
GET /health/status (returns branded HTML)

KV Store

Session-scoped key-value storage for edge logic:

POST /api/v1/kv?token=YOUR_TOKEN
{ "key": "feature_flag", "value": "enabled" }

GET /api/v1/kv?token=YOUR_TOKEN&key=feature_flag

DELETE /api/v1/kv?token=YOUR_TOKEN&key=feature_flag
CLI REFERENCE

zyppr tunnel

zyppr tunnel <port> [ports...] [flags]

Flags:
  --tcp             TCP tunnel
  --udp             UDP tunnel
  --auth user:pass   Password protection
  --server host:port Custom server
  --token TOKEN      Override saved token
  --no-reconnect     Disable auto-reconnect

zyppr up

zyppr up [--config path/to/zyppr.yml]

# Start all tunnels defined in zyppr.yml

zyppr p2p

zyppr p2p --peer <peer-id> --local-port <port>

Flags:
  --peer ID         Peer identifier
  --local-port PORT   Local port to forward
  --server host:port   Signaling server

zyppr auth

zyppr auth <token>

# Save authentication token to ~/.zyppr/config
