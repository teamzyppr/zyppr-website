Frequently Asked Questions

Everything you need to know about Zyppr, our protocols, and features.

Zyppr supports HTTP, HTTPS, TCP, and UDP protocols. This covers web servers, databases (PostgreSQL, MySQL), SSH, RDP, game servers, IoT devices, DNS forwarding, VoIP, and any custom protocol. All HTTP tunnels are automatically secured with TLS certificates.
Yes! Zyppr is the only tunneling platform that supports UDP tunneling. You can tunnel game servers (Minecraft on port 25565), DNS (port 53), VoIP, IoT sensor data, and any UDP-based protocol using zyppr tunnel 25565 --udp or the zero-download bridge.
SSH method: Zero-download. Works on any machine with a terminal. Run a standard SSH command and your tunnel is live. No binary to install. Supports HTTP and TCP.

CLI binary: Download the zyppr binary for the fastest experience. Uses QUIC transport (UDP-based, faster than TCP), supports HTTP/TCP/UDP, auto-reconnect, config-as-code, and P2P mesh.
Extremely secure. Every tunnel uses end-to-end encryption with cryptographic verification. Additionally, our AI Gateway detects SQL injection, XSS, specific paths, and suspicious activity in real-time. You can also add password protection, rate limiting, and IP-based blocking.
Yes! On the Pro (Beam) plan and above, you can bring your own domain. Simply add a CNAME record pointing to our server, and Zyppr automatically provisions and auto-renews a Let's Encrypt TLS certificate.
P2P Mesh connects two machines directly via UDP hole punching, bypassing the server entirely. Our server acts as a signaling mechanism to exchange NAT information. This results in near-zero added latency for direct connections.
Chaos Engineering lets you inject faults into your tunnel to test your application's resilience. You can add artificial latency (e.g., 500ms), configure a percentage of requests to return errors (e.g., 5% return 500), or simulate network issues.
Webhook breakpoints pause an incoming HTTP request mid-flight. When enabled, the next request hitting your tunnel is held. You can inspect the full request (headers, body, method), modify it, and then release it to your server.
DB Studio is a built-in web-based SQL browser for your tunneled databases. You can connect to PostgreSQL, MySQL, or SQLite databases running on your localhost, browse schemas and tables, and execute read-only SQL queries from your UI.
